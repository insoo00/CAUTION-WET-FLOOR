# Handoff Notes — Claude.ai 대화 요약

이 문서는 Claude Code에서 새 세션을 시작할 때 컨텍스트 보충용입니다.
CLAUDE.md를 보강하는 역할.

## 대화에서 결정된 사항

### 음원 소스 결정
- **애플 뮤직 직접 연동은 제외**: MusicKit JS는 DRM 때문에 Web Audio 분석 불가 + Apple Developer Program 가입 비용($99/년) 부담
- **현재 채택**: YouTube IFrame API
  - 영상 ID 예시: `Ik9hLZsHU7g` (사용자가 연습 중인 "나는 나비" 영상)
- **향후 추가 예정**: 본인 소유 mp3 업로드 (Web Audio API 풀 액세스로 정밀 분석/속도 조절)

### 동기화 전략 결정
"YouTube와 악보를 어떻게 정밀하게 맞출 것인가" 문제에 대해:
- **자동 score following (마이크/오디오 분석)** — 정확하지만 구현 복잡, 향후 Phase 5
- **수동 매핑 (현재 채택)** — 사용자가 곡 재생 중 마디 시작점을 5~6개 탭으로 찍어두면 선형 보간으로 그 사이를 계산
- 5~6개 포인트면 곡 전체가 거의 정확하게 따라간다는 게 핵심 아이디어
- 매핑은 곡당 한 번만 하면 됨 → localStorage(현재) → IndexedDB(Phase 2)

### 자동 스크롤 전략 결정
- **페이지 단위 넘김 (기존 PDF 악보 앱들)** — 페이지 경계에서 끊김 발생
- **연속 스크롤 (현재 채택)** — Blackbinder 앱과 같은 접근. 커서가 뷰포트 하단 80px 근처면 부드럽게 스크롤. 페이지 경계 자체가 없음.

### MusicXML 우선 결정
- **PDF 악보 + 마디 수동 매핑** — 모든 악보 받지만 정확도 낮음
- **MusicXML (현재 채택)** — 음표/마디 정확하게 추적 가능. 사용자 곡은 MuseScore.com 커뮤니티에 대체로 있음. PDF는 향후 폴백 옵션으로.

## 사용자 컨텍스트
- 피아노 초보 (밴드에서 건반 담당)
- 박자감이 약한 편 → 리듬 패턴 시각화가 중요
- 휴대성 위해 iPad에서 사용 예정
- 비개발자 가능성 있음 → 친절한 설명 + 실행 명령어 명시 필요
- 한국어로 응답

## 첫 메시지 추천 패턴

Claude Code 시작 시 권장 첫 메시지:

```
@CLAUDE.md @docs/roadmap.md @prototype/band-practice-v2.html 보고
프로젝트 상태 파악해줘.

그 다음 Phase 1 시작하자:
1. 먼저 어떻게 진행할지 계획을 보여주고
2. 내 확인 받고 나서 코드 작성 시작
```

`@파일경로`로 파일을 명시하면 Claude Code가 명시적으로 그 파일을 읽고 시작합니다.

## 자주 쓸 명령어

- `claude` — 인터랙티브 세션 시작 (현재 폴더 기준)
- `claude --resume` — 마지막 세션 이어서
- 세션 안에서 `/clear` — 컨텍스트 초기화 (다른 작업 시작할 때)
- 세션 안에서 `/help` — 명령어 목록
